ACCIDENTS常見事故

營運 AI 團隊必知的六大深坑

這些都是真實發生過的事故，提前了解，能幫您省下可觀的額度與無形的時間成本

1. Token 金鑰外洩

API Key 誤傳到 Github 公開倉庫，一小時被盜刷 $50 美元。

2. 長對話變笨 (塌陷)

歷史對話過長，超出 Context 限制，導致 Agent 邏輯錯亂、忘記規則。

3. 自動化誤觸發

未對測試環境隔離，測試自動化流程時，在 production 環境觸發了真動作。

4. 同 Bug 重複死鎖燒錢

Agent 在同一個 bug 上因重複 debug 陷入無限循環，燒乾 API 額度。

5. 部署到錯誤專案

從錯誤的目錄跑部署， deploy 到了不該 deploy 的正式環境專案。

6. 制度自己違反自己

制定了規則但自己沒遵守（如說好不 git add . 但還是手動加了）。

🛡️ 第一防線：自動化敏感詞掃描
每一次 git commit 之前，本地自動觸發密鑰掃描器（如 trufflehog3）。一旦檢測到疑似金鑰的亂碼，立刻中斷提交並發送警報。

嚴禁任何包含真實個資的檔案直接進入 Git 或公開 API。必須經過「個資去敏過濾器」將敏感個資替換為去敏代號，並在最後發布前經過 Jacob 的 LINE 人工確認閘門。
