STRATEGY省錢大法

三大省錢策略：Token 消耗銳減 90%

不盲目呼叫最高級的 S 級模型，善用機械檢查與便宜模型分工

💻 機械檢查交給腳本 (0 Token)
核心原理：排版、語法檢查、檔案是否存在等動作，用本機寫好的 Shell/Node 腳本來跑，完全不需要呼叫 LLM。驗收通過後，才放行進 LLM 進行語意分析，省下大量無謂的 token 開銷。
⚖️ 模型分級混用 (省下 80%)
核心原理：大量的文件粗加工、資料格式化，指派給便宜的模型（如 GPT-4o-mini 或 Gemini Flash，B 級）；只有最後的關鍵技術決策與語意把關，才交給最聰明的模型（如 Claude 3.5 Sonnet，S 級）。

🛡️ 第一防線：自動化敏感詞掃描
每一次 git commit 之前，本地自動觸發密鑰掃描器（如 trufflehog3）。一旦檢測到疑似金鑰的亂碼，立刻中斷提交並發送警報。

嚴禁任何包含真實個資的檔案直接進入 Git 或公開 API。必須經過「個資去敏過濾器」將敏感個資替換為去敏代號，並在最後發布前經過 Jacob 的 LINE 人工確認閘門。
