# 🔄 任務狀態機 — 一人版小卡

> 每個任務都走這個流程，一眼看出「卡在哪裡」。

---

## 狀態流程

```
📋 已指派 → [/] 執行中 → [✅ 完成]
                ↓
            [📤 移交]（交給下游）
                ↓
            [⏸️ 掛起]（等外部條件）
                ↓
            [❌ 取消]
```

---

## 狀態說明

| 狀態 | 意思 | 誰負責 |
|------|------|--------|
| 📋 已指派 | 任務寫好了，等人接手 | 你（派工的人） |
| [/] 執行中 | 有人正在做 | 執行者 |
| ✅ 完成 | 做完了，驗收通過 | 你驗收 |
| 📤 移交 | 做完交棒給下一個人 | 移交者 |
| ⏸️ 掛起 | 卡住了，等外部條件 | 記錄原因 |
| ❌ 取消 | 不做了 | 你決定 |

---

## 鐵則

- 每個任務只能有一個 `[/] 執行中` 的持有者
- 看到 `[/]` 或 `📤` 就停手，不搶
- 卡住立刻寫原因，不等別人問
