# 🎯 派工三件套模板 — 一人版

> 派工給 AI 時，這三樣缺一不可。少了任何一項，AI 就會亂猜方向。

---

## 三件套清單

### 1️⃣ 目標與動機（做什麼＋為什麼）

```
目標：在 {檔案/目錄} 實作 {功能}。
動機：{這是給誰用的、解決什麼問題——邊界情況時照這個方向猜}。
```

**為什麼重要**：沒有動機，AI 遇到模糊選擇時會亂猜方向。

---

### 2️⃣ 驗收條件（怎麼判斷做完了）

```
驗收條件（全過才算完成）：
- {具體行為：輸入 X 得到 Y}
- {機器驗證：跑指令通過}
- 檔案存在且內容非空
```

**鐵則**：
- 禁止抽象詞：「完善」「高品質」「適當」一律不行
- 要能 grep、能跑、能數
- 做的人不驗自己

---

### 3️⃣ 回報格式（做完後回報什麼）

```
回報要求：
- 只回結論，不要貼原文
- 引用位置用 檔案路徑:行號
- 明確區分「確認的事實」和「推測」
- 若中途發現任務前提有誤，停下來回報
```

---

## 範例：完整派工 prompt

```
目標：在 `docs/` 目錄建立一份 API 文件。
動機：新進工程師需要快速了解 API 端點，不用翻原始碼。
背景：遵循現有 `docs/api-design.md` 的格式。

驗收條件：
- 檔案 `docs/api-reference.md` 存在
- 包含 GET/POST/PUT/DELETE 四個端點說明
- 每個端點有請求/回應範例
- 跑 `grep -c "^## " docs/api-reference.md` 結果 ≥ 4

回報要求：改了哪些檔、驗收逐條 pass/fail、未解決事項。
```
