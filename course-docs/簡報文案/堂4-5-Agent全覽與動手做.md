## 第 4 堂：M3 Agent 全覽 — 選對的工具

### 頁 1：封面
**標題**：Agent 全覽 — 選對的工具
**要點**：
- 不是聊天機器人，是有記憶、會學習、能執行的助理
- 市面 Agent 百百種，選錯工具 = 浪費時間

### 頁 2：Agent 是什麼
**標題**：Agent ≠ 聊天機器人
**要點**：
- ChatGPT 只能對話，每次開新對話從零開始
- Agent 有記憶（跨對話記得你）、能執行指令（不只說還會做）、能連接外部工具（串 API、讀檔案、跑腳本）
- 想像：聊天機器人是「問答機」，Agent 是「隨身助理」

### 頁 3：Agent 的三大能力
**標題**：有記憶、會執行、能串接
**要點**：
- **記憶**：记住你的偏好、專案進度、上次做到哪裡
- **執行**：不只是回答問題，能幫你改檔案、跑腳本、發訊息
- **串接**：連接 Telegram/Discord/Notion/n8n 等外部服務
- 三者加起來 = 真正的「助理」，不是「百科全書」

### 頁 4：市面常見 Agent 比較
**標題**：四大 Agent 一次看
**要點**：
- **Hermes**：隨身助理，Telegram/Discord 介面，記憶＋排程，適合個人日常
- **Codex**：工作流助理，終端機介面，執行＋自動化，適合開發者
- **OpenCode**：遠端執行，雲端環境，適合需要伺服器的場景
- **n8n**：自動化中樞，視覺化工作流，串接多個服務，適合進階用戶

### 頁 5：功能比較表
**標題**：誰能做什麼？
**要點**：

| 能力 | Hermes | Codex | OpenCode | n8n |
|------|--------|-------|----------|-----|
| 記憶 | ✅ | ✅ | ✅ | ❌ |
| Telegram/Discord | ✅ | ❌ | ❌ | 可串 |
| 跑 code | ❌ | ✅ | ✅ | 有限 |
| 自動化排程 | ✅ | ❌ | ❌ | ✅ |
| 視覺化介面 | ❌ | ❌ | ❌ | ✅ |

### 頁 6：如何選擇
**標題**：依需求選 Agent
**要點**：
- 個人日常（記事、提醒、問問題）→ Hermes（Telegram/Discord）
- 開發自動化（改 code、跑測試、部署）→ Codex + OpenCode
- 多服務串接（自動發通知、同步資料）→ n8n
- 預算有限 → 先從免費方案開始（Hermes 免費、Codex 有限免費額度）

### 頁 7：組合策略
**標題**：不只選一個
**要點**：
- 一人起步：Hermes 就夠（日常助理 + 記憶）
- 有客戶後：加上 n8n（自動化通知 + 流程）
- 有產品後：加上 Codex（代碼審查 + 部署）
- 原則：先求有、再求好，不要一次開五個 Agent

### 頁 8：實做 — 決策表
**標題**：動手做時間
**要點**：
- 填寫你的需求（日常/開發/自動化）
- 依裝置選擇（手機/電腦/伺服器）
- 依預算選擇（免費/付費）
- 產出：你的 Agent 選擇決策
- 學員帶走：`Agent選擇決策表.md`

### 頁 9：小結
**標題**：M3 小結——選工具的三步
**要點**：
- **先問痛點**：你最花時間的事是什麼？
- **再問場景**：你在手機上還是電腦上用？
- **最後問預算**：能花多少錢？
- 下堂課：動手做你的第一個 Agent

## 第 5 堂：M4 動手做 — 你的第一個 Agent

### 頁 1：封面
**標題**：動手做 — 你的第一個 Agent
**要點**：
- 今天你會完成 Hermes 安裝
- 跟你的 AI 助理說第一句話

### 頁 2：今天的目標
**標題**：完成三件事
**要點**：
1. 選擇你的介面（Telegram 或 Discord）
2. 完成 Hermes 安裝與部署
3. 跟你的 AI 助理說第一句話並確認記憶正常

### 頁 3：部署選擇
**標題**：Telegram vs Discord
**要點**：
- **Telegram**：推薦一般用戶，簡單快速，手機直接用
- **Discord**：推薦家庭教育，可建私人伺服器，多人协作
- 兩者都支援語音、圖片、檔案傳輸
- 選擇建議：一個人用 → Telegram；家庭/團隊用 → Discord

### 頁 4：安裝前準備
**標題**：需要準備什麼
**要點**：
- 一組 API Key（從 AI 服務商取得）
- Zeabur 帳號（免費額度夠用）
- 10 分鐘時間
- 準備好了嗎？開始！

### 頁 5：Hermes 安裝 Step 1
**標題**：建立 Bot
**要點**：
- Telegram：找 @BotFather → /newbot → 取名 → 拿到 Token
- Discord：Developer Portal → New Application → Bot → 拿到 Token
- 把 Token 存好（等下要用）

### 頁 6：Hermes 安裝 Step 2
**標題**：Zeabur 部署
**要點**：
- 進 Zeabur → 匯入 Hermes 模板
- 貼上 Token（環境變數）
- 一鍵部署（等 30 秒）
- 驗證：看到 "Bot is running" 就成功了

### 頁 7：Hermes 安裝 Step 3
**標題**：首次啟動
**要點**：
- 在 Telegram/Discord 找到你的 Bot
- 發送 `/start`
- 它會回你一句問候
- 確認：它記得你（問「你記得我嗎？」）

### 頁 8：防失憶設定
**標題**：Volume 設定
**要點**：
- 為什麼需要：容器重啟會丟資料（記憶消失）
- 如何設定：Zeabur 掛載路徑 `/root/.hermes`
- 驗證：重啟後問「你記得我嗎？」→ 如果記得就成功
- 忘了設定 = 每次重啟都要重新自我介紹

### 頁 9：安全設定
**標題**：白名單保護
**要點**：
- 為什麼需要：防止別人用你的 Bot（燒你的額度）
- 設定方式：環境變數 `ALLOWED_USERS=你的ID`
- 找到你的 ID：在 Telegram 跟 @userinfobot 說話
- 驗證：用另一個帳號測試 → 被拒絕 = 設定成功

### 頁 10：基本功能測試
**標題**：玩玩看
**要點**：
- 問問題：「今天天氣如何？」
- 傳圖片：「幫我描述這張圖」
- 記憶測試：先說「我叫小明」→ 過一陣子問「我叫什麼？」
- 檔案功能：傳一份文件請它摘要

### 頁 11：常見問題
**標題**：卡住了怎麼辦
**要點**：
- Bot 不回話 → 檢查 Token 是否正確、ALLOWED_USERS 有沒有設錯
- 記憶消失 → 檢查 Volume 有沒有掛載
- 回覆很慢 → 正常，免費方案有速率限制
- 錯誤訊息 → 貼到 Google 搜尋，90% 的問題都有人遇過

### 頁 12：進階：記憶管理
**標題**：讓它記住更重要的事
**要點**：
- 你可以主動告訴它「請記住：我的客戶叫XXX，喜歡XXX」
- 它也會自動記住你常問的東西
- 定期清理：太久以前的記憶可能不準確
- 安全：敏感資料（密碼、API Key）不要讓它記

### 頁 13：實做 — 完成安裝
**標題**：動手做時間
**要點**：
- 依選擇的介面完成部署
- 與你的 AI 助理說第一句話
- 確認記憶功能正常
- 學員帶走：`Hermes開工.md`、`Hermes收工.md`

### 頁 14：小結
**標題**：M4 小結——你的第一個 Agent 上線了
**要點**：
- **安裝三步**：建 Bot → Zeabur 部署 → 首次對話
- **防失憶**：Volume 掛載，重啟不丟記憶
- **安全**：白名單，只有你能用
- 下堂課：實戰案例——建立你的 Discord 隨身書僮
