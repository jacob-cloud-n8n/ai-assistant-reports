## 第 7 堂：M6 實戰案例 — 五人團隊

### 頁 1：封面
**標題**：實戰案例 — 五人團隊
**要點**：
- 多 Agent 協作的真實運作
- 今天你會畫出你的 Agent 團隊架構圖

### 頁 2：五人團隊架構
**標題**：誰做什麼？
**要點**：
- Hermes：監督者（接收指令、派工、驗收）
- Codex：編譯手（讀規格、改 code、部署）
- OpenCode：執行者（跑指令、做體力活）
- AG：規格腦（只出規格，不碰 code）
- n8n：自動化中樞（串接服務、觸發工作流）

### 頁 3：協作模式
**標題**：狀態機流程
**要點**：
- 📋 已指派 → [/] 執行中 → [✅ 完成]
- 每個任務只能有一個持有者
- 看到 `[/]` 或 `📤` 就停手，不搶

### 頁 4：DoD 驗收機制
**標題**：做的人不驗自己
**要點**：
- 驗收條件必須可 grep/可跑/可數
- 禁止抽象詞：「完善」「高品質」
- 機器先跑全過才進語意驗收

### 頁 5：防呆規則
**標題**：單向指令流
**要點**：
- ✅ Jacob → Agent（指令）
- ✅ Agent → 團隊專區（狀態更新）
- ❌ 直接呼叫其他 Agent
- ❌ 未經驗收就宣告完成

### 頁 6：實做 — 畫架構圖
**標題**：動手做時間
**要點**：
- 繪製你的 Agent 團隊
- 定義各 Agent 的角色
- 學員帶走：`團隊專區模板.md`、`狀態機小卡.md`

## 第 8 堂：M7 從使用者到管理者

### 頁 1：封面
**標題**：從使用者到管理者
**要點**：
- 今天你會學 Cron 排程、記憶管理、收工報告
- 設定一個定時任務

### 頁 2：Cron 排程
**標題**：讓 AI 自己按時做事
**要點**：
- 定時任務的概念（像鬧鐘）
- 設定提醒（每天 09:00 開工提醒）
- 自動化工作流（每天 07:00 讀板頭推日報）

### 頁 3：記憶管理
**標題**：什麼該記、什麼不該記
**要點**：
- 該記：客戶偏好、專案進度、重要決策
- 不該記：臨時草稿、敏感資訊、已過期資料
- 記憶清理的安全流程（備份→清理→驗證）

### 頁 4：收工報告
**標題**：每日收工 SOP
**要點**：
- 更新團隊專區狀態
- 記錄今天做了什麼
- 標記明天要做的事
- 自動生成日誌

### 頁 5：實做 — 設定定時任務
**標題**：動手做時間
**要點**：
- 建立一個 Cron 排程
- 測試自動執行
- 學員帶走：`派工三件套模板.md`、`DoD檢核卡.md`
