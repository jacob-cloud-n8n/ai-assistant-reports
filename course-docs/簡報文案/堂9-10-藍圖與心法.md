## 第 9 堂：M8 Agent 團隊藍圖

### 頁 1：封面
**標題**：Agent 團隊藍圖
**要點**：
- 今天你會規劃你的 AI 團隊
- 完成你的 Agent 團隊規劃畫布

### 頁 2：為什麼需要藍圖
**標題**：有方向才不會迷路
**要點**：
- 很多人的問題：裝了很多工具，但不知道要幹嘛
- 藍圖的作用：從痛點出發，選對工具，規劃產線
- 你不需要五個 Agent——你需要一個能解決你痛點的 Agent

### 頁 3：規劃方法論
**標題**：從需求出發
**要點**：
- 先問「你有什麼痛點？」再問「需要什麼 Agent？」
- 漸進式擴展：先求有、再求好
- 不要一次開五個 Agent——從一個開始
- 原則：能解決痛點的工具才是好工具

### 頁 4：痛點盤點
**標題**：你最花時間的事是什麼？
**要點**：
- 列出你每天花時間的 5 件事
- 標記哪些是「重複性高」的（自動化候選）
- 標記哪些是「需要判斷」的（AI 助理候選）
- 標記哪些是「需要多人协作」的（團隊候選）

### 頁 4B：痛點分類表
**標題**：三種痛點，三種解法
**要點**：

| 痛點類型 | 特徵 | 解法 | 範例 |
|---------|------|------|------|
| 重複性高 | 每天做、步驟固定 | 自動化腳本（n8n） | 每日報表生成 |
| 需要判斷 | 有規則但需人工決策 | AI 助理（Hermes/Codex） | 客戶回覆分類 |
| 需要协作 | 多人多步驟 | 團隊 + 看板 | 課程交付流程 |

### 頁 5：畫布工具
**標題**：產線規劃畫布
**要點**：
- **你有什麼需求？**（痛點清單，按優先級排）
- **你需要什麼 Agent？**（工具選擇，對照痛點）
- **你願意投入多少？**（時間＋金錢，量力而為）
- **每條產線** = owner + 流程 + 模板
- 範例：一張填好的畫布

### 頁 5B：畫布填法示範
**標題**：一張填好的畫布
**要點**：

| 欄位 | 填法 | 範例 |
|------|------|------|
| 痛點 | 你最花時間的事 | 每天花 1 小時回覆客戶詢問 |
| Agent | 能解決這個痛點的工具 | Hermes（Telegram AI 助理） |
| 流程 | 觸發 → 處理 → 輸出 | 客戶傳訊 → AI 分類 → 自動回覆 |
| 模板 | 重複使用的 prompt/腳本 | 客戶問答模板.md |
| 時間 | 每天花多少時間維護 | 每週 30 分鐘檢查 |
| 預算 | 每月花多少錢 | API 費用 $5-20/月 |

### 頁 6：現金流視角
**標題**：你的 Agent 要接你自己的生意
**要點**：
- 用「產線」思維：每條產線有 owner、流程、模板
- 範例 1：**羊奶訂閱配送**——n8n 自動接單 → 通知牧場 → 安排配送
- 範例 2：**收納服務**——LINE 表單接案 → 自動排程 → 完工通知
- 範例 3：**家教班**——報名表 → 自動分班 → 課前提醒
- 原則：先穩固一條產線，再擴展第二條

### 頁 6B：產線案例深挖
**標題**：羊奶訂閱配送——從零到自動化
**要點**：
- **痛點**：每天 manual 回覆 20+ 則訂單訊息
- **Agent**：n8n 工作流 + LINE 官方帳號
- **流程**：客戶傳「訂羊奶」→ n8n 解析 → 自動回覆確認 → 通知牧場 → 排配送
- **結果**：從每天 1 小時降到 5 分鐘（只處理例外）
- **教訓**：先跑通一週，確認零錯誤再上線

### 頁 7：產線三階段
**標題**：穩固 → 擴展 → 規模化
**要點**：
- **穩固**：一條產線跑通、零錯誤、不需人工介入
- **擴展**：加上第二條產線，共用同一套 Agent
- **規模化**：多條產線並行，開始帶團隊
- 你現在在哪？大多數人還在「穩固」第一條

### 頁 7B：三階段案例
**標題**：從收納服務到三條產線
**要點**：
- **穩固（第 1 個月）**：LINE 表單接案 → 自動回覆 → 完工通知
- **擴展（第 2 個月）**：加上 Google 日曆自動排程 → 不用手動排
- **規模化（第 3 個月）**：加上 Notion 客戶管理 → 自動發滿意度調查
- **關鍵**：每階段跑順再進入下一階段，不要跳級

### 頁 8：避開常見陷阱
**標題**：不要踩這些坑
**要點**：
- ❌ 一次開五個 Agent → 每個都半吊子
- ❌ 先選工具再想需求 → 工具不對，重來
- ❌ 追求完美 → 先上線再優化
- ❌ 忽略安全 → API Key 外洩一次就虧大

### 頁 8B：陷阱案例
**標題**：真實踩坑故事
**要點**：
- **案例 1**：同時開五個 Agent → 每個都沒跑通 → 一個月後全部關掉
- **案例 2**：先裝 n8n 再想需求 → 不知道要幹嘛 → 放著沒用
- **案例 3**：追求完美不敢上線 → 三個月後還在 test
- **教訓**：先求有、再求好、再求快

### 頁 9：實做 — 完成規劃
**標題**：動手做時間
**要點**：
- 填寫你的產線規劃畫布
- 定義你的第一條產線（owner + 流程 + 模板）
- 制定你的下一步（本週要完成什麼）
- 學員帶走：`產線規劃畫布v2.md`

### 頁 10：小結
**標題**：M8 小結——藍圖的三個關鍵
**要點**：
- **從痛點出發**：先問痛點，再選工具
- **一條產線原則**：先穩固一條，再擴展
- **量力而為**：時間和預算決定你的起步規模
- 下堂課：AI 閱讀三大心法——用 AI 加速學習

## 第 10 堂：M9 AI 閱讀三大心法

### 頁 1：封面
**標題**：AI 閱讀三大心法
**要點**：
- 狩獵法（納瓦爾）——帶問題讀資料
- 費曼法（費曼）——用自己的話解釋
- 辯論法（蒙格）——找不同觀點辯論

### 頁 2：為什麼需要心法
**標題**：有工具還不夠，要有方法
**要點**：
- 很多人的問題：裝了 AI 但不知道怎麼用
- 有心法 → 知道什麼時候用、怎麼用、用到什麼程度
- 三大心法對應三個場景：快速消化、確認理解、決策思考

### 頁 3：狩獵法
**標題**：帶問題讀資料
**要點**：
- 核心：不求讀完，只求破局
- 步驟：先問「我想解決什麼問題？」→ 讀資料 → 找到答案就停
- 適合場景：快速消化大量資料、找特定答案
- 範例：帶着「怎麼降低 AI 月費？」的問題讀成本優化文章

### 頁 4：狩獵法實戰
**標題**：用 AI 做狩獵
**要點**：
- 提示詞：「我有一個問題：XXX。幫我從這份資料中找到答案。」
- AI 會自動過濾不相關的內容，只給你關鍵資訊
- 找到答案就停，不要貪多
- 技巧：問題越具體，答案越精準

### 頁 4B：狩獵法完整案例
**標題**：帶着問題讀 STAR 57 筆記
**要點**：
- **問題**：「我的課程要怎麼定價？」
- **做法**：把 57 筆記丟給 AI，問「哪些筆記有提到定價策略？」
- **結果**：AI 找出 3 筆相關內容（S5-價值包裝、A10-名單磁鐵、R1-業績模型）
- **你做什麼**：只讀這 3 筆，不碰其他 54 筆
- **省多少**：從 2 小時→20 分鐘

### 頁 5：費曼法
**標題**：用自己的話解釋
**要點**：
- 核心：讓 AI 反問你，確認你真的懂
- 步驟：讀完 → 試着解釋給「不懂的人」聽 → 解釋不清 = 你還沒懂
- 適合場景：確認理解、準備教學、深度學習
- 範例：讀完「狀態機」後，試着解釋給 AI 聽，讓它挑毛病

### 頁 6：費曼法實戰
**標題**：用 AI 做費曼
**要點**：
- 提示詞：「我剛讀完 XXX，請用費曼法考我——我解釋完後你反問我細節。」
- AI 會像老師一樣追問你：「為什麼是 3 次不是 2 次？」
- 如果你答不出來 → 回去重讀
- 技巧：讓 AI 扮演「不懂的人」，你越解釋越清楚

### 頁 6B：費曼法完整案例
**標題**：用費曼法學「止損三次」
**要點**：
- **你讀完**：止損三次卡.md
- **你解釋**：「就是 AI 同一個 bug 試三次就要停下來」
- **AI 追問**：「三次是指三次嘗試，還是三次不同方法？」
- **你發現**：你沒讀清楚——是「單次派工 2 輪停手、任務累計 3 次凍結」
- **教訓**：費曼法會暴露你以為懂但其實沒懂的地方

### 頁 7：辯論法
**標題**：找不同觀點辯論
**要點**：
- 核心：讓 AI 扮演不同立場，幫你多角度思考
- 步驟：提出你的想法 → 讓 AI 扮演正方 vs 反方 → 找出最佳方案
- 適合場景：重大決策、架構選擇、投資判斷
- 範例：「用 Firebase 還是 Zeabur？」→ 讓 AI 扮演兩方辯論

### 頁 8：辯論法實戰
**標題**：用 AI 做辯論
**要點**：
- 提示詞：「我想做 XXX 決策。請分別扮演支持者和反對者，各給三個論點。」
- AI 會給你正反兩面的論據
- 你自己做最終判斷——AI 是顧問，不是決策者
- 技巧：追問「還有第三個觀點嗎？」往往能得到意想不到的視角

### 頁 8B：辯論法完整案例
**標題**：選工具的辯論
**要點**：
- **決策**：用 Firebase 還是 Zeabur 部署官網？
- **正方（AI 扮演）**：Firebase 免費額度高、全球 CDN、整合 Google 服務
- **反方（AI 扮演）**：Zeabur 台灣速度快、支援 Docker、部署更簡單
- **第三觀點**：看你團隊熟悉什麼——如果已經會 Docker 就選 Zeabur
- **你做決定**：選 Zeabur（因為團隊已經在用）
- **教訓**：AI 給選項，你做決定

### 頁 9：三法整合
**標題**：什麼時候用什麼法
**要點**：

| 場景 | 推薦心法 | 範例 |
|------|---------|------|
| 快速找答案 | 狩獵法 | 讀文件找解法 |
| 確認理解 | 費曼法 | 準備教學、深度學習 |
| 做決策 | 辯論法 | 選工具、選方案 |
| 學新東西 | 狩獵→費曼 | 先找重點，再確認懂 |

### 頁 9B：心法組合技
**標題**：三法連環使用
**要點**：
- **場景**：你要決定「要不要買一台伺服器」
- **步驟 1（狩獵法）**：快速找出相關資訊——價格、比較、替代方案
- **步驟 2（費曼法）**：解釋給 AI 聽「我為什麼想買」，讓它挑毛病
- **步驟 3（辯論法）**：讓 AI 扮演正反方，幫你做最終判斷
- **結果**：你做出 informed decision，不是衝動購買

### 頁 10：實做 — 心法體驗
**標題**：動手做時間
**要點**：
- 選一篇文章（或你的專案文件）
- 用狩獵法找三個關鍵答案
- 用費曼法解釋其中一個給 AI 聽
- 用辯論法為一個決策找正反論點
- 學員帶走：`三大心法提示詞.md`

### 頁 11：小結
**標題**：M9 小結——三大心法的共同點
**要點**：
- **主動**：不是被動接收，是帶着目的去用 AI
- **互動**：不是單向提問，是跟 AI 來回對話
- **驗證**：不是照單全收，是確認自己真的懂
- 下堂課：踩坑紀錄——這些錯誤你不能犯
