PLATFORM平台優勢

為什麼大流量團隊首選 Discord？

相較於 Telegram，Discord 在安全邊界、協作效率與權限管理上具備跨代優勢

1. 免綁手機門號：Telegram 強制與個人門號綁定，且有嚴格的帳號數量限制；Discord 開通僅需電子信箱，極利於團隊部署多個功能助理。

2. 群組與頻道架構：支援在單一伺服器下開設多個主題頻道（例如記帳頻道、代辦頻道、測試頻道），讓不同 Agent 在各自的頻道內各司其職，資訊不亂流。

3. 企業級權限管理：可以對不同成員、Bot 設定極為精細的角色與白名單，確保敏感指令不被誤觸發。

WORKFLOW三步部署

從零到一，Zeabur 雲端一鍵部署

透過 Developer Portal 取得憑證，再到 Zeabur 部署執行，告別繁複的本機守護進程設定

Step 1: 建立 Discord Bot

前往 Discord Developer Portal 建立 Application，在 Bot 頁面點擊 "Reset Token" 取得密鑰，並在 OAuth2 開通相關 Bot 權限。

Step 2: Zeabur 一鍵部署

在 Zeabur 平台點擊 "Add Service"，選擇 Git Repository 或者是直接使用 Hermes 的預設模板快速拉起服務。

Step 3: 設定環境變數

在 Zeabur 的 Variables 區設定 `DISCORD_BOT_TOKEN` 以及 `DISCORD_ALLOWED_USERS` 白名單，重啟服務即可開機。

PERSISTENCE持久記憶

防失憶關鍵：Zeabur 持久化 Volume 設定

沒有 Volumes，雲端容器每一次重建（Deploy/Rebuild）都會導致您的 Agent 徹底失憶

❌ 裸容器重建 (無 Volume)
致命痛點：Docker 容器是無狀態的。一旦您更新代碼、重啟服務，本地儲存的對話歷史、客戶記憶會被全部抹除。Agent 重開後會問「您是誰？」，工作資產徹底歸零。
✅ 掛載 Persistent Volume
正確做法：在 Zeabur 的 "Destinations / Volumes" 頁面中新增一個磁碟掛載，指定路徑為 `/root/.hermes`。這樣不論容器如何重啟重建，記憶文件依然完好無損。

白名單機制：把安全威脅阻斷在外

設定白名單與防越權機制，防止惡意第三方通過您的 Bot 盜用您的付費 Token 額度
