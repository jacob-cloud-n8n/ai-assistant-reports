## 模組詳情

### M0：AI 素養與環境準備

| 項目 | 內容 |
|------|------|
| **你會學到** | 怎麼跟 AI 說話、環境準備、安全防護與權限控制、AI 協作自律禮儀 |
| **實做項目** | 申請 OpenRouter 帳號、防禦性專案初始化（Git + .gitignore 配置）、安裝 Hermes、第一個「開工/收工」儀式 |
| **學員帶走** | `AGENTS.md` 工作規則模板、防禦性 `.gitignore` 模板、特助權限安全指引小卡、開工/收工 SOP 小卡 |
| **時間** | 60 min（可延長至 90 min） |

**教學內容：**
1. 設備與軟體準備
   - Mac / Windows / Linux 皆可
   - 終端機基本操作
   - 瀏覽器設定
2. 帳號與金鑰安全準備
   - Telegram 帳號與 API 金鑰申請
   - OpenRouter API Key 的安全管理與防洩漏觀念
   - Zeabur 帳號、GitHub 帳號（可選）
3. AI 特助安全防線與協作禮儀
   - 提示詞公式：角色 + 任務 + 格式 + 限制
   - 角色設定技巧與追問引導
   - **安全防線：特助的命令執行權限與半自動（Semi-automatic）確認機制**
   - **AI 協作自律禮儀：為什麼需要「開工與收工」？**
   - **單一真相來源 (SSOT) 觀念**：防範 AI 失憶與上下文塌陷，重要產出沉澱至本地
4. 實做：防禦性專案初始化與蘇格拉底費曼法體驗
   - 專案目錄初始化 `git init`，並立刻配置防洩漏的 `.gitignore`（過濾 `.env` 及 Cookie）
   - 結合開工儀式，讓 AI 擔任思維教練反問挑戰，重新理解 AI 助理
   - 體驗以 AI 當思維教練，而非被動接受答案的威力

### M1：AI 的失憶症與對話解耦

| 項目 | 內容 |
|------|------|
| **你會學到** | 為什麼每次都要重講一遍、上下文視窗 (Context Window) 原理、對話解耦心法、模型 S/A/B 分級 CP 值分析 |
| **實做項目** | 記錄你每天重複對 AI 說的話，規劃你的模型分工與對話分流 |
| **學員帶走** | `痛點自檢.md`、`模型分級對流卡.md` |
| **時間** | 30 min（可延長至 60 min） |

**教學內容：**
1. AI 的失憶問題與底層原理
   - ChatGPT 每次開新對話從零開始，工作流無法累積
   - 底層解析：什麼是 Token？什麼是 Context Window（上下文視窗）？
   - **上下文塌陷 (Context Collapse) 機密：為什麼對話越長，AI 會變笨並丟失設定？**
2. 對話解耦與上下文管理心法
   - **對話解耦（Decoupling）**：一個對話只專心做一件事，防範資訊雜訊交叉污染
   - **對話壓縮與寫回**：當特助變笨時，如何命令它將精華壓縮為 SSOT 寫回本地，並開啟全新對話重新載入
3. 為什麼需要不一樣的工具與模型分工
   - 跨對話的長效記憶系統的重要性
   - **模型 S/A/B 分級與省錢大作戰**：用便宜模型（B級）做程式粗工與檔案檢索，把聰明模型（S級）留給頂層規劃與 QA 驗收，Token 效率最大化
4. 實做：痛點自檢與時間/成本試算
   - 列出你每天重複對 AI 說的話，試算一年浪費的時間與 Token 費用
   - 規劃你的對話分流地圖

### M2：建立你的第二大腦與說明書分級

| 項目 | 內容 |
|------|------|
| **你會學到** | 三層結構（輸入→消化→輸出）、資料自主、全域與專案說明書分級記憶 |
| **實做項目** | 建立 Obsidian 資料夾 + 貼上全域/專案 CLAUDE.md |
| **學員帶走** | `CLAUDE.md` 模板、資料夾結構圖、記憶分級實踐指引 |
| **時間** | 45 min（可延長至 60 min） |

**教學內容：**
1. 第二大腦概念
   - Karpathy 的 AI 筆記流
   - 資料不被平台綁架，你的知識資產自己掌控
2. 三層結構
   - Clippings/（輸入：別人的文章）
   - 知識庫/（消化：AI 幫你整理）
   - 創作庫/（輸出：你的原創內容）
3. 同步與安全備份機制
   - GitHub 私有 repo
   - Google Drive 備份
   - 跨裝置、跨 Agent 同步
4. 說明書的分級與獨立治理
   - **記憶與說明書分級**：什麼是全域說明書 (Global Context, 如 `~/.config/` 規則)？什麼是專案說明書 (Project Context, 如 `.agents/` 下的 `CLAUDE.md`)？
   - **讀取優先權機制**：當 AI 特助一開工，它如何逐層讀取這些規則？如何利用專案說明書覆蓋全域規則，實現多專案獨立治理與精準控制。
5. 實做：建立資料夾結構與說明書部署
   - 建好三層資料夾
   - 部署專案專屬的 `CLAUDE.md` 工作規則，並讓 Agent 執行開工測試

### M3：Agent 全覽：選對的工具

| 項目 | 內容 |
|------|------|
| **你會學到** | 各 Agent 的特色、適合場景、選擇方法 |
| **實做項目** | 用決策表選出適合你的 Agent |
| **學員帶走** | `Agent選擇決策表.md` |
| **時間** | 30 min（可延長至 60 min） |

**教學內容：**
1. Agent 是什麼
   - 不是聊天機器人
   - 有記憶、會學習、能執行
2. 市面常見 Agent 比較
   - Hermes：隨身助理，記憶+排程
   - Codex：工作流助理，執行+自動化
   - OpenCode：遠端執行
   - n8n：自動化中樞
3. 如何選擇
   - 依需求選 Agent
   - 依裝置選 Agent
   - 依預算選 Agent
4. 實做：決策表
   - 填寫你的需求
   - 找出最適合的 Agent 組合
