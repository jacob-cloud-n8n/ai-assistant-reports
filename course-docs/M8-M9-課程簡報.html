HUNTING METHOD狩獵法

狩獵法：帶有極強目的性的「精準狙擊」

不求把資料從頭到尾讀完，只求尋找解決當前局面的那一個「解藥」

1. **獵物定位**：開始閱讀前，先對 Agent 下達明確的狩獵指令。例如：「在這份 300 頁的開車法規中，我只想找出『酒駕罰款與吊扣扣分條款』」。

2. **拿到答案就停**：找到關鍵點，立刻退出，拒絕被其餘無關的背景雜訊和理論衍生帶走注意力。

3. **破局高於完整**：商場如戰場，解決問題的速度比追求學術上的「讀完」更有價值。

強制停止條件明確告知：找到解答，立即結束對話

FEYNMAN METHOD費曼法

費曼法：用您自己的話，解釋給小學生聽

如果不能用最白話的中文解釋清楚一個概念，就說明您還沒有真正理解它

❌ 傳統的「自以為懂了」 ✅ 費曼法引導交互學習
讀完一份 Docker 教學，點點頭覺得邏輯通暢，但自己開終端機寫不出半行命令。 讓 Agent 扮演「不懂技術的小孩」，要求你用最直白的譬喻（如把 Docker 比喻成貨櫃）向它解釋。
把文案從頭到尾死記硬背，但一旦遇到客戶提問，無法靈活拆解或重組。 讓 Agent 在你解釋完之後，反過來針對「漏洞」對你進行追問與挑戰，直到毫無死角。

DEBATE METHOD辯論法

辯論法：激發 AI 的對立人格做壓力測試

做重大商業或技術決策前，命令 AI 扮演反對者角色，對您的方案進行無情的漏洞挑選

🔵 辯論對立設置
1. **正方 (您或支持 AI)**：列舉該方案的所有優勢。
2. **反方 (挑刺 AI)**：無情指責技術風險、成本超標與安全漏洞（如：憑證暴露風險）。
⚖️ 最終辯證與決策
透過正反兩方 2-3 輪的激烈辯論，最脆弱的假設會被撕碎，逼著您在真正執行前，拿出最防呆的備用方案（如：Volumes 持久化）。
