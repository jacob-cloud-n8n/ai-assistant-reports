DEFINITION定義

Agent ≠ 聊天機器人

聊天機器人只是「問答機」，Agent 則是擁有記憶、能連外部工具並能執行指令的「隨身助理」

傳統的 ChatGPT 只負責對話，每次開新視窗都是一次「記憶清空」的從頭來過，極難累積工作資產。

而真正的 Agent（智慧代理） 能夠：

1. 本地端儲存持久化記憶，不怕對話變笨失憶。

2. 直接呼叫終端機、資料庫或 API 幫您實際幹活。

3. 串接多個系統，實現全自動的接單、編譯與發布。

Agent 的三大核心能力
持久記憶庫不怕長對話失憶，工作資產可沉澱
工具連接器能實際執行代碼、訪問網頁與資料庫
自主執行流依據工作說明書，自動完成複雜任務

INTERFACE介面選擇

Telegram 還是 Discord？選一個最合適的

雙介面皆支援語音、圖片與檔案，您的偏好決定了助理的「棲息地」

📱 Telegram：快速實用、說走就走
優勢：開通極快，僅需 1 分鐘申請 Bot Token。手機上直接用語音、簡訊進行即時溝通、發送 PDF 進行文件摘要。非常適合個人工作流程與行動派發任務。
💻 Discord：多人協作、安全權限管理
優勢：無須綁定個人手機號碼。可為家庭或公司建立專屬的私人伺服器，將不同的 Bot 依頻道分組，具備極為精細的白名單與權限過濾機制，適合多人與多 Bot 協同運作。

Step 1: 下載與初始化腳本

開啟本機終端機，執行 npm 初始化腳本安裝 Hermes 核心依賴。

Step 2: 配置 API Key 與啟動

在產生的 .env 設定檔中，貼入您的金鑰並運行 start 指令。

# 寫入環境變數並啟動服務
npm run start

ONBOARDING首次開機

與您的 AI 助理說第一句話

驗證基本對話、語音辨識與持久化記憶，確保助理的大腦已完全開通

1. 發送 `/start` 啟動，Hermes 會回傳開工歡迎語並向您報告它的目前技能。

2. 測試 **語音或圖片傳輸**。比如發一張您的名片，讓它辨識並萃取文字。

3. 測試 **持久化記憶**：告訴 it「記住我的羊奶宅配客戶是高先生」，接著關閉重新啟動，再次詢問「我的羊奶客戶是誰？」，檢驗其 Volume 記憶是否完好。
