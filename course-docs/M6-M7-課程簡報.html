協作核心：單一任務的狀態流轉

為了防止多個 AI 特助搶寫同一個檔案，全團隊必須嚴格遵守非同步狀態機

1. [📋 已指派]

任務寫入 `團隊專區.md`，此時為靜態，尚無特助認領執行。

2. [/] 執行中

單一特助認領並標記為執行中。看到這個標記，其餘特助必須立刻停手，嚴禁搶單。

3. [✅ 完成]

經由下游特助驗收通過，標記完成，將控制權還給老闆。

單向指令流：Agent 互動的鐵律

嚴禁 Agent 之間直接互叫，所有通訊與任務移交必須通過唯一真相來源：團隊專區

✅ 允許的指令流向
1. 老闆 ➔ Agent：下達指令與需求變更。
2. Agent ➔ 團隊專區：更新自身任務狀態與完成報告。
3. Agent ➔ 老闆：提報阻塞（Block）與等待決策。
❌ 絕對禁止的行為
1. Agent 直接呼叫 Agent：這會引發無窮循環、燒毀 API 額度。
2. 未經檢驗的「自吹自擂」：未經 Codex 跑機器測試就自我宣稱任務已完成。
3. 直接改動 Production 環境：跳過驗收程序。

CLOSE-OUT REPORT收工規範

每日收工 SOP：為明天留下清晰的指引

好的收工日誌，能讓明早開工的 Agent 在 10 秒內接棒，無縫展開工作

1. **更新專區狀態**：將手中已完成的任務標記為 [✅ 完成]，未完的明確寫出阻塞點。

2. **掃描敏感金鑰**：在 commit 之前，利用工具掃描是否有密鑰外流。

3. **只 Git Add 相關檔案**：嚴禁盲目執行 git add .，避免將暫存的垃圾檔案一併提交。

4. **自動生成日誌**：執行收工腳本，自動整理本日異動並推送至遠端。

收工日誌必填三要素
本日已做完什麼條列式說明，附上對應的 commit hash
明日優先待辦為明早開工的自己或特助留下精確指令
當下卡關點標記黃燈與阻塞對象，主動預警
