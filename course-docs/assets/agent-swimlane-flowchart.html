多 Agent 協作狀態機與 DoD 流程

Multi-Agent Workflow & Definition of Done (DoD)

💬 Claude (任務編排) 🛠️ OpenCode (工兵實作) 🔍 Codex (品質把關) 🚀 AG (全局打包)
1. [📋 已指派]
任務規劃與指派
Claude 理解人類需求，將大任務拆解並指派，建立獨立的 .tasks/<task-id>.md 檔案。
2. [/] 執行中
工兵實作開發
OpenCode 在獨立分支上執行代碼修改，並完成本地測試、Lint 檢查。
3. [📤 移交]
建立 PR 與前置報告
實作完成後，OpenCode 產出 Pre-review Report 並建立 PR，移交給 Codex 審核。
4. [/] 驗證中
DoD 技術審查官
Codex 進行安全掃描、自動化 E2E 測試與程式碼 Review。若通過則簽署 Codex Review = accepted，否則退回。
5. [✅ 完成]
二腦同步與 Commit / Push
AG 執行二腦增量編譯，並進行 git diff 安全二次確認，統整並 commit 到 main 分支關閉任務。
