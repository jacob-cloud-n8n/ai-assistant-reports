## 試算表欄位結構與公式設計

如果您希望在自己的 Excel 或 Notion Database 中手動重建這個試算表，請參考以下結構設計與公式：

### 1. 欄位明細

| 欄位名稱 | 類型 | 說明 | 範例數據 |
| :--- | :--- | :--- | :--- |
| **A. 重複性溝通情境** | 文字 | 日常與 AI 溝通時，每次都要手動貼入的背景或要求。 | 「每次都要貼入我的自我介紹與寫信語氣要求」 |
| **B. 每天重複次數** | 數字 | 該情境平均每天會發生幾次。 | `5` |
| **C. 每次貼入的背景字數** | 數字 | 每次對話手動貼入的參考資料、代碼背景或長 Prompt 字數。 | `500` |
| **D. 每日浪費時間 (分鐘)** | 公式 | 每次複製貼上與調整格式所花費的時間。 | `7.5` *(計算公式見下)* |
| **E. 每月浪費 Token** | 公式 | 每月該重複背景在對話中被 LLM 讀取的 Token 總數。 | `97,500` *(計算公式見下)* |
| **F. 建議優化方案** | 判斷公式 | 依據浪費程度自動判斷最適合的 AI 設定工具。 | `Project CLAUDE.md` *(判斷邏輯見下)* |

### 2. 計算公式 (Excel / Sheets 適用)

*   **D. 每日浪費時間 (分鐘)**：
    *   *設定*：每次複製、貼上並調整 Prompt 估計花費 1.5 分鐘。
    *   *公式*：`=B2 * 1.5`
*   **E. 每月浪費 Token**：
    *   *設定*：以繁體中文 1 字約折合 1.3 個 Token 計算，每個月以 30 天計。
    *   *公式*：`=B2 * C2 * 1.3 * 30`
*   **F. 建議優化方案 (自動判定判斷)**：
    *   *邏輯*：
      - 若每月浪費 Token < 30,000 ➔ 建議使用「**系統說明書 (Global Context / Custom Instructions)**」
      - 若每月浪費 Token 在 30,000 ~ 150,000 ➔ 建議使用「**專案說明書 (Project CLAUDE.md)**」
      - 若每月浪費 Token > 150,000 ➔ 建議將其「**封裝為獨立 AI Skill (以代碼控制)**」
    *   *公式*：`=IF(E2<30000, "系統說明書 (Global Context)", IF(E2<=150000, "專案說明書 (CLAUDE.md)", "封裝成 AI Skill"))`

## 💡 常見優化方案實作建議

1.  **系統說明書 (Global Context / 自訂指令)**：適合全域通用的規則。例如「請一律以台灣繁體中文回答，不使用中國大陸用語」。
2.  **專案說明書 (CLAUDE.md)**：適合特定資料夾/專案。例如「當我在 `/Project/Marketing` 資料夾下，請自動扮演 FB 廣告文案小幫手，並帶入行銷受眾人設」。
3.  **封裝成 AI Skill**：適合有複雜外部工具調用或大批量處理的需求。例如「每次丟入 PDF 發票，自動辨識明細並寫入本地 Google 試算表」。
