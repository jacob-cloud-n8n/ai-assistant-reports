🎓 AI Agent 實戰家教課程

為孩子建立專屬的
「隨身書僮」 (Hermes)

針對無手機號碼的學齡兒童，以 Discord 為互動介面，手把手部署專屬的 Hermes AI 助理，並設定物理隔離的私有知識庫與防失憶機制，建立安全、受大人監管的學習伴侶。

開始實戰課程

🔑 免手機註冊

Telegram 註冊必須綁定真實手機門號。學齡兒童通常沒有門號，而 Discord 允許純電子信箱（Email）註冊與登入。

📂 物理隔離知識庫

每位孩子擁有專屬的 GitHub 私有儲存庫作為大腦。她們的個人筆記、記憶與學習進度完全隔離，不與其他人混雜，安全私密。

💾 Volume 防失憶

在雲端服務重啟或版本更新時，透過掛載持久化 Volume，保證機器人的長期記憶不丟失，伴隨孩子長大。

「孩子拿到手機平板，總是點開遊戲或抖音，怎麼可能拿來讀書？」

🎯 1. 轉娛樂為學習利器

這不是教孩子「寫程式碼」的技術課，而是一門重塑學習習慣、培養自主提問與邏輯思辨的思維引導課。我們教導孩子將科技設備轉化為專屬的「學習秘書」，而非遊戲玩具。

結合「上學囉！/放學囉！」的每日儀式感，搭配實體習慣養成小卡，引導孩子在讀書前自主拆解任務、設定計畫，課後主動對話備份。將抽象的自主管理化為日常物理習慣。

帳號由家長共管，孩子不需要專屬手機。Bot 僅在封閉、乾淨的頻道中互動，並具備強大的敏感個資保護與不良內容過濾機制。對話歷史完整存入 GitHub，資訊透明且安全防失憶。

第一階段：無手機門號申請 Discord 帳號

學齡兒童沒有專屬門號時，如何在大人的監督下順利註冊並通過認證？

🚨 實戰踩坑 1：Discord 的「13歲年齡限制限制」

問題：若註冊時填寫真實生日且年齡小於 13 歲，帳號會立刻被官方系統鎖定封鎖。
解法：家教上課時，請大人填寫大於 13 歲的年份（或由家長親自代持該帳號），以順利完成註冊。

  • ✓ 準備專屬 Email：使用 Gmail 或利用大人的備用 Email 註冊。
  • ✓ 以信箱而非手機註冊：點選信箱註冊，並至信箱中點擊「驗證電子信箱」。

🚨 實戰踩坑 2：未驗證 Email 會使伺服器邀請「顯示過期」

問題：如果帳號未點擊信箱驗證信，加入伺服器時會彈出「邀請連結已過期/無效」的誤導性提示。
解法：加入頻道前，必須 100% 確保 Email 驗證已通過。

第二階段：為孩子建立隨身書僮端到端部署

家長跟著帶練的 5 大實戰技術步驟。

Step 1 在 GitHub 上建立專屬私有知識庫 (KB-Vault)

🔧 點擊查看 Step 1 詳細操作步驟與代碼指令

使用 GitHub CLI 建立孩子的私有 Repo。這將作為她專屬的記憶與筆記庫。

# 建立大女兒的私有知識庫
gh repo create jacob-cloud-n8n/hermes-junior-kb --private --add-readme

# 建立小女兒的私有知識庫
gh repo create jacob-cloud-n8n/hermes-senior-kb --private --add-readme

Step 2 在 Discord Application Portal 申請 Bot Token

🔧 點擊查看 Step 2 詳細操作步驟與權限勾選設定

進入開發者後台，註冊 Bot 並開啟必要的對話讀取權限。

  • 1. 登入 Discord Developer Portal 點擊 New Application。
  • 2. 點選左側 Bot ➔ 點擊 Reset Token 取得 DISCORD_BOT_TOKEN。
  • 3. ⚠️ 核心開關：限制開啟 Presence、Server Members、與 Message Content Intent (沒開 Bot 會讀不到訊息！)。
  • 4. 點選左側 OAuth2 -> URL Generator，Scopes 勾選 bot、applications.commands，並在權限勾選發送、讀取、上傳檔案，用產出的連結把 Bot 邀請進伺服器。

Step 3 在 Zeabur 部署並配置專屬服務

🔧 點擊查看 Step 3 詳細一鍵部署指令與變數設定

切換至本機專案部署目錄，使用 CLI 一鍵部署至雲端，並設定物理隔離環境變數。

cd "/Users/jacob/Projects/2026 hermes/deploy/hermes-agent"
npx zeabur deploy --project-id <專案ID> --create --name hermes-junior --interactive=false

在 Zeabur 控制台環境變數（Environment Variables）中配置以下關鍵安全變數：

  • DISCORD_BOT_TOKEN = (步驟二獲取的 Bot Token)
  • KB_VAULT_REPO = jacob-cloud-n8n/hermes-junior-kb.git (指向步驟一建立的私有庫)
  • GITHUB_PAT = (您的 GitHub PAT) (用於 Bot 推送更新)
  • DISCORD_ALLOWED_USERS = 您的DiscordID (先填寫您的 ID，以便進行測試)

Step 4 掛載持久化磁碟 (Volume) 以防重啟忘記記憶

🔧 點擊查看 Step 4 詳細持久磁碟掛載路徑與步驟

避免雲端容器重新啟動或部署時，本地的 SQLite 資料庫遺失。

  • 1. 開啟 Zeabur 控制台 ➔ 點入剛部署好的服務頁面。
  • 2. 找到 磁碟 (Volumes) 區塊，點擊新增磁碟。
  • 3. 掛載路徑 (Mount Path) 必須填寫為： /root/.hermes。

Step 5 頻道綁定與加入孩子的白名單防禦

🔧 點擊查看 Step 5 詳細頻道綁定與 Allowed Users 設定

將 Bot 鎖定在指定的聊天頻道，並將孩子加入白名單，防止非白名單成員惡意發話呼叫。

  • 1. 在預計對話的 Discord 頻道中輸入 /sethome 進行主頻道綁定。
  • 2. 複製孩子的 Discord 使用者 ID (需在 Discord 設定 ➔ 進階 ➔ 開啟「開發者模式」，對頭像點右鍵複製)。
  • 3. 修改 Zeabur 環境變數 DISCORD_ALLOWED_USERS 為 您的ID,孩子的ID (以英文逗號分隔)，儲存並重啟。

📅 5 堂實戰課程大綱

1

第一堂：認識 AI 與個資安全防線

了解 AI 的能力與幻覺，學會數位素養與隱私防護，建立合規帳號與完成書僮初始化命名。

2

第二堂：Prompt 指令基本功與追問

掌握與書僮對話的「角色、限制、格式」三大要素，並熟練與 AI 連續追問修正的互動技巧。

3

第三堂：三大學習心法實務體驗

實作體驗「狩獵法（提煉重點）、費曼法（說給書僮聽）、辯論法（思辨挑戰）」的學習魔力。

4

第四堂：自主規劃大目標拆解

用 AI 書僮將大科目的段考範圍拆解成每日學習計畫，實作「上學囉！/放學囉！」的每日存檔 SOP。

5

第五堂：成果展示與習慣卡建立

展示自主規劃成果，發放「實體習慣養成卡」，教導使用指南，設定家長週報與後續追蹤計畫。
