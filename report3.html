❌ 傳統無助理模式

重複、瑣碎、容易疲憊的日常

  • ✕ 手動謄寫工單：客戶來單需人工複製到 Google Sheets 與 Notion。
  • ✕ 資訊查找耗時：需不斷在歷史聊天紀錄中查找客戶歷史預約與偏好。
  • ✕ 頻繁的手動流程：開工需要手動執行程式庫更新、收工需要人為篩選金鑰與撰寫成果日誌。
  • ✕ 資訊孤島：各平台（LINE、Discord、Notion）數據散落，無法連動。
✅ AI 助理協作模式

自動、自癒、高效流暢的體驗

  • ✓ 全自動資料歸檔：LINE 訊息或表單進來，AI 自動結構化並同步寫入多端資料庫。
  • ✓ 即時上下文彙整：AI 助理能自動攜帶上下文，將客戶背景資料整理好提供給管理員。
  • ✓ 一鍵代理指令：對助理說「開工」或「收工」，系統自動接管 Git 拉取、金鑰掃描過濾、推播日誌。
  • ✓ 錯誤捕獲與自癒：工作流出現錯誤時自動重試，並將異常日誌即時推送到 Discord 通知。

⚡ 「開工與收工」全自動化代理流程

在我們的專案開發中，我們不手動處理冗雜的日常行政。透過 Codex 與 Hermes 助理，我們的協作方式如下：

1. 一鍵開工

開發者下達「開工」指令，Codex 自動拉取遠端最新代碼並執行 git rebase，同時讀取 ANTIGRAVITY 專案規範與駕駛艙筆記。

2. 資安與過濾

AI 在開發過程中隨時監控，嚴防將 API keys、Google Cookie 等敏感金鑰 commit 至公開 repo。

3. 自動收工推送

下達「收工」指令，助理自動掃描變更、撰寫 walkthrough 成果報告，並自動 stage 本次相關檔案並推送到 GitHub。
