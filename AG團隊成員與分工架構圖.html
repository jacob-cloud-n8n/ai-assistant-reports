Jacob's Business OS

AG Agent 團隊與專案架構藍圖

四層架構運算模型。滑鼠移入任何一個卡片，即可自動突顯其上下游關聯專案、指派對象與所屬工具，關係一目了然。

💡 操作提示：請將滑鼠指標懸停在任意卡片上以探索其關聯脈絡。
01
Layer 1: 決策與互動層
Jacob (雅各老師 / 決策者)
系統的最高指令發起人。透過團隊專區派發任務，接收每日自動化晨報，並擁有 Obsidian 專案駕駛艙。
主導核心
學員 / 家人 (如：愛恩)
一線系統的使用者。透過 Discord / Telegram 接收隨身助理的學習陪伴與知識檢索服務。
服務對象
Hermes / 四兒 (隨身書僮)
一線互動端 Agent。提供手機端 (TG/Discord) 陪伴、知識庫檢索與防失憶 Volume 自動備份。
一線助理
02
Layer 2: 協同中樞與 AI 引擎
n8n 核心流程中樞
自動化神經中樞。定時讀取任務、寫入 Notion 數據庫，並分析日報發送至 Telegram。
流程中樞
Notion CRM & 任務看板
任務快照與客戶 CRM 資料庫。與 n8n 雙向同步，作為團隊日常任務的實體管理看板。
任務看板
GitHub Private KB-Vaults
物理隔離的私有知識庫與程式碼倉庫。儲存一線對答歷史，確保敏感變數安全不外洩。
安全儲存
AntiGravity (AG)
架構分析與 UI 設計 Agent。負責大局架構設計、銷售網頁視覺拋光與合約 PDF 自動編譯。
架構規劃
Codex
代碼重構與測試 Agent。專注於本地 Rust/TS (tolaria) 測試健康度，以及 n8n JS 代碼開發。
程式與測試
OpenCode / MiMo
自動部署與營運 Agent。負責 Zeabur 雲端環境變數配置與 Astro 5 網站自動化部署。
自動部署
03
Layer 3: 工作區專案地圖
1. Open Design 專案
Vite + PostgreSQL 設計專案，由 AI 陣容進行代碼維護與庫結構設計。
專案維護
2. meidi-home-site
美地居家收納官網 (Astro 5 + Zeabur SSR)，資料由 Notion 同步，OpenCode 負責自動部署。
官網建置
3. Skills 共享技能庫
團隊共享 Markdown 技能包。Agent 透過 CLI 一鍵下載，達到技能快速同步與複用。
技能包
4. Star 專案 (範本)
React + Vite 快速開發範本。作為新專案初始化的腳手架。
開發範本
5. Tolaria App
Tauri 2 + Rust + React 本地知識庫 App。由 Codex 負責本地單元測試健康度把關。
軟體重構
6. 進銷存系統
React 19 庫存系統。由 Hermes 追蹤收集客戶反饋，Codex 進行核心修正。
系統優化
7. 威力馬方案
經銷商教育訓練方案。內建 RAG 常見問答與引流漏斗，由 Hermes 提供一線對答支援。
訓練支援
8. 銷售專案 (Slide)
靜態 HTML 簡報與銷售網頁。由 AG 進行 UI 微動效拋光，並部署至 Firebase。
銷售包裝
04
Layer 4: 技術與工具箱
Obsidian
雅各老師的人工專案駕駛艙。不連接自動化插件，保留純粹個人反思與任務指派。
個人隨身庫
NotebookLM
Google 科學文獻與長文本分析大腦。用來輔助 Jacob 進行深度學術研究與大綱消化。
研究大腦
Firebase Hosting
靜態網頁與簡報託管服務。銷售專案的發布目的地，具備安全與極速 CDN 特性。
部署託管
Playwright / Chrome
合約 PDF 自動渲染轉換器。呼叫 Headless 瀏覽器將 Markdown 客製化文件完美轉換為列印格式。
文件轉檔
開啟課程文件資料夾
