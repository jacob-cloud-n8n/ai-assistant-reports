# Systeme.io 課程上架文案 (模組 1 – 模組 4)

本文件彙整「AI 助理運用與實戰課程」模組 1 至模組 4 之上架文案，提供課程平台 (Systeme.io) 上架配置之用。

## 模組 1：AI 素養與個人特助環境準備

### 1. 模組課程描述
本模組專為 AI 零基礎的學習者設計，旨在培養必備的 AI 素養並完成個人特助的開發環境準備。課程將帶領您理解 AI 協作的思維轉變，從最基礎的 OpenRouter API 金鑰安全申請、配置 Git 防火牆防範金鑰洩漏，到實行 AI 協作的開工與收工黃金自律禮儀。學員亦將透過「學習護照」與 AI 教練互動，量身打造五級學習階梯，為後續的助理開發奠定最穩固的安全與思維基石。

### 2. 每課 (Lesson) 標題與簡介
*   **課 1：OpenRouter 金鑰申請與安全防禦**  
    學習如何申請本線上課程的核心 API 金鑰 (OpenRouter Key)，並於本地專案中建立 `.gitignore` 安全防線，徹底隔絕金鑰被誤傳上 GitHub 的潛在風險。
*   **課 2：AI 協作自律禮儀：開工與收工**  
    揭密與 AI 高效協作的黃金自律儀式。透過「開工對齊」Prompt 載入教練人設，以及「收工總結」將知識轉存為 Markdown 小卡，拒絕對話沙灘化。
*   **課 3：學習護照體驗：費曼法對抗與學習階梯**  
    實操如何與 AI 教練進行費曼法對話練習，並輸入引導 Prompt 讓 AI 針對您的程度與具體目標，量身產出專屬的「五級學習階梯」護照。

## 模組 2：克服 AI 健忘症與打造 Obsidian 第二大腦

### 2. 模組課程描述
當與 AI 進行長對話時，往往會遇到 AI 記憶模糊或忘記脈絡的「健忘症」痛點。本模組將教導您利用本地 Markdown 筆記軟體 Obsidian 打造個人的「三層結構第二大腦」(Clippings/Brain/Projects)。您將學習如何撰寫專案說明書 `CLAUDE.md`，讓 AI 助理每次開工時一秒對齊脈絡。同時，我們將手把手教您如何連結 GitHub 私有倉庫，在 3 秒內完成二腦的雲端加密備份，確保知識資產永不遺失。

### 2. 每課 (Lesson) 標題與簡介
*   **課 1：Obsidian 實戰：打造你的三層結構第二大腦**  
    引導安裝 Obsidian 並建立「01 收集、02 知識、03 專案」三層經典資料夾，演示如何藉由雙向連結建立知識網絡，並在 Projects 中置入關鍵的 `CLAUDE.md`。
*   **課 2：3分鐘極速設定：使用 GitHub 私有倉庫備份你的第二大腦**  
    手把手教學如何建立 GitHub Private 倉庫，並透過終端機指令將本地二腦與雲端私有庫連結，實現 3 秒極速加密同步與雲端防護。

## 模組 3：Agent 全覽與部署你的第一個 AI 助理

### 3. 模組課程描述
本模組為您剖析 AI Agent (智慧代理) 的核心架構與市面主流平台 (Coze, Dify) 的優缺點。接著，我們將以 Nous Research 的 Hermes 助理為例，帶您實操如何在雲端 Zeabur 服務上一鍵部署屬於您的 24 小時 Telegram 智慧特助。本模組會著重解決「雲端重啟遺忘記憶」與「金鑰被他人盜刷」兩大痛點，引導您配置雲端 Volume 掛載持久化以及用戶白名單防禦變數。

### 2. 每課 (Lesson) 標題與簡介
*   **課 1：Telegram 機器人申請與安全 Token 管理**  
    學習如何在 Telegram 搜尋 @BotFather 並利用指令快速申請機器人 Token，強調金鑰安全與保管原則。
*   **課 2：Zeabur 雲端服務一鍵部署與環境變數綁定**  
    實操如何關聯 GitHub 與 Zeabur 控制台，匯入程式碼並綁定 API Token，在 1 分鐘內讓您的機器人助理正式在雲端運作。
*   **課 3：進階配置：Volume 持久化儲存與白名單防禦**  
    學習在 Zeabur 新增硬碟 Volume 並掛載到指定目錄，解決容器重啟失憶問題；設定允許的使用者白名單變數，將不速之客拒之門外。

## 模組 4：多 Agent 協作與個人時間審計系統

### 4. 模組課程描述
本模組將帶您進入 AI 團隊的多 Agent 協同作戰領域，深入剖析 AI 助理們 (AG, Codex, OpenCode, Hermes) 的職責邊界與任務狀態機。課程將教導您機器可判定的驗收標準 (DoD) 原則，並手把手教學如何使用 Docker Compose 在本地端拉起完全免費且無限次執行的 n8n 工作流中樞。最後，您將導入經典的「柳比歇夫時間統計法」，透過手機端 Webhook 傳送記錄，由 AI 自動拆解分類並即時寫入 Notion，實現數據驅動的個人營運與商業估時決策。

### 2. 每課 (Lesson) 標題與簡介
*   **課 1：多 Agent 團隊分工與任務狀態機**  
    深入了解不同 AI 角色 (如 Codex 審查、OpenCode 開發、AG 統籌) 在專案中的定位，以及任務如何沿著「已指派➔執行中➔移交➔完成」狀態機流轉。
*   **課 2：機器可判定驗收標準 (DoD) 原則**  
    學習摒棄模糊不清的驗收口氣，制定編譯成功、測試 100% 通過、網頁 200 回應與無斷鏈等機器可客觀驗證的 DoD 門檻。
*   **課 3：本地自動化沙盒：n8n Docker-Compose 啟動與基礎操作**  
    學習如何在 Docker Desktop 啟用狀態下，透過終端機執行一行指令部署本地 n8n 與資料庫，建立自己專屬的無上限自動化沙盒。
*   **課 4：時間審計系統實戰：從時間記錄到 Notion 自動彙整**  
    導入 `lyubishchev-workflow.json` 工作流，使 Webhook 捕獲手機發送的記事，並呼叫 AI 智能解析，單向投影至 Notion，精算出開發專案的「純專注時間」與「AI 時間垃圾」。
