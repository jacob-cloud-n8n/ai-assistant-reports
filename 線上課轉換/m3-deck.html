第一單元：常見 Agent 工具與平台

什麼是 AI Agent？

  • 主動規劃 (Planning)：能將複雜任務拆解成多個子步驟並自主執行，而非單純一問一答。
  • 記憶系統 (Memory)：包含短期對話上下文，以及與 Volume/資料庫對接的長期記憶。
  • 工具調用 (Tools)：能呼叫網頁搜尋、API、資料庫讀寫等外部工具擴展能力邊界。

主流 Agent 平台對比

平台類型 優勢 劣勢 適用場景
Coze / Dify 介面友善、工作流畫布強大、支援多平台一鍵發佈 資料隱私與自訂度受限，平台限制較多 快速驗證原型與常規工作流
Hermes / 自部署 100% 掌握代碼、自由度極高、可自訂安全防範機制 需要基本的雲端部署概念與環境配置 個人/團隊專屬的安全隱私助理

第二單元：BotFather 申請 Telegram Bot

步驟：在 Telegram 建立新機器人

1. 開啟 Telegram，搜尋並點擊官方的 @BotFather
2. 發送指令開始建立：
/newbot
3. 依提示輸入機器人的暱稱 (例如：我的隨身特助)
4. 輸入唯一的 Username (必須以 bot 結尾，如 my_assistant_bot)
5. 複製生成的 HTTP API Token (例如：123456789:AAFfakeToken...)
⚠️ 安全鐵律：Bot Token 代表該機器人的最高控制權，請妥善保管，千萬不可截圖公開或上傳到 GitHub 公開倉庫！

第三單元：Zeabur 雲端服務一鍵部署

為什麼選擇 Zeabur？

  • 免管理伺服器：不需懂得 Linux 指令或設定複雜的 SSL 憑證，一鍵完成部署。
  • GitHub 連動：綁定個人 GitHub 專案，當程式碼 push 更新時，Zeabur 會自動觸發構建與更新。
  • 輕鬆配置變數：透過視覺化介面設定環境變數，將敏感 API Key 與程式碼物理抽離。

部署時的環境變數綁定

在 Zeabur 服務的 Variables 設定頁面新增以下項目：

變數名稱 (Key) 說明 範例值
TELEGRAM_BOT_TOKEN 您的 Telegram 機器人金鑰 123456789:AAFfakeToken...
OPENROUTER_API_KEY OpenRouter API 金鑰 sk-or-v1-your-key-here...
HERMES_INFERENCE_PROVIDER LLM 供應商設定 deepseek

第四單元：Volume 持久化與安全防護

雲端硬碟 (Volume) 與白名單防護

Volume 持久化

解決重啟失憶。因為雲端容器是唯讀臨時的，必須手動在 Zeabur 建立硬碟掛載至 /app/data，確保記憶 JSON 檔永久保存。

白名單限制

防範 API 金鑰被他人盜刷。設定白名單變數，限制僅有指定用戶 ID 可以跟機器人說話。

Volume 與白名單配置範本

# 限制僅允許的使用者 ID (多個人用逗號隔開)
TELEGRAM_ALLOWED_USERS=123456789,987654321

# 指定記憶體檔案的持久化儲存路徑
MEMORY_FILE_PATH=/app/data/memory.json
💡 如何獲取個人 Telegram ID？
在 Telegram 搜尋 @userinfobot 並發送隨意訊息，它會立刻回覆您的個人數字 ID (例如 123456789)。
