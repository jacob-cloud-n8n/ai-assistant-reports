第一單元：多 Agent 分工與角色職責

多 Agent 團隊：為什麼需要分工？

  • 防範模型能力瓶頸：沒有單一模型能完美做好所有事，分工能讓特定 Agent 專精特定任務。
  • 降低 Token 浪費：讓低成本的助理處理常規工作，高智商審查者進行最終把關。
  • 提高專案穩定度：透過角色物理與權限隔離，避免核心程式被不相干的 Prompt 意外破壞。

AI 助理團隊成員與分工職責

Agent 角色 核心職責 允許動作 禁止動作
AG (AntiGravity) 架構設計、UI 拋光、統整提交 設計網頁、編譯 PDF、git commit/push 治理文件 大規模修改核心邏輯代碼
Codex 技術審查、測試把關、程式碼重構 代碼審查、合併 PR、直接修復 Bug 與重構 未跑測試即合併 PR 至 main 分支
OpenCode / MiMo 原型實作、自動部署與測試 實作功能代碼、本機測試、建立 PR 分支 直接 merge 至 main，裸部署生產環境
Hermes (四兒) 一線陪伴、客戶問答、文件整理 對話陪伴、更新 Notion 與 docs/、發送報告 存取核心代碼、進行生產部署

第二單元：任務狀態機與 DoD

任務生命週期：黃金狀態機

# 任務流轉狀態
[📋 已指派] ➔ [/] 執行中 ➔ [📤 移交] (Codex 審核) ➔ [✅ 完成]
  • 一人一任務：任務在同一時間只能有一個持有者，防止協作衝突。
  • 去中心化任務紀錄：每個任務獨立建檔於 .tasks/<task-id>.md，拒絕共用單一文件。

機器可判定驗收標準 (DoD)

Definition of Done (DoD) 必須是機器可判定的硬指標，拒絕含糊

  • 專案編譯 (Build) 成功，無任何嚴重錯誤與 Lint 警報
  • Unit Test 與 E2E 測試 100% 通過，無 regression
  • 指定網頁 URL HTTP 狀態碼為 200 (運作正常)
  • verify-links.sh 執行無任何斷鏈與 404
  • 最終審查關卡：Codex Review 標記為 accepted

第三單元：本地自動化沙盒 n8n

為什麼在本地使用 Docker 啟動 n8n？

  • 無限制使用：雲端免費版有執行次數限制，本地 Docker 則是 100% 免費且無限次運行。
  • 資料隱私安全：工作流與資料庫均儲存在本地硬碟，保護個人與商業隱私不外洩。
  • 一鍵啟動：透過 docker-compose.yml 配置文件，一行指令拉起 n8n 與 PostgreSQL。

本地 n8n 啟動與操作

# 1. 電腦安裝並開啟 Docker Desktop
# 2. 來到工作目錄，在終端機輸入：
docker compose up -d

# 3. 檢查容器是否正常啟動：
docker compose ps

# 4. 瀏覽器造訪本地畫布，完成初始帳號註冊：
http://localhost:5678
💡 參數說明：-d 參數代表背景執行 (Detached mode)。啟動後，即使關閉終端機，n8n 也會繼續永久運作。

第四單元：時間審計系統實戰

什麼是柳比歇夫時間統計法？

  • 以數據對抗主觀偏見：對自己「純專注時間」(Net Time) 進行無情、誠實的紀錄。
  • 定量 AI 運算消耗：記錄 AI 助理在 Rate Limit 或死鎖 Debug 重試花費的時間與 Token，精算真實成本。
  • 數據驅動報價決策：分析「主業開發」、「輔業維護」及「時間垃圾」，作為未來估時的科學指標。

時間記帳自動化數據流

1. Webhook 觸發器

透過手機快捷指令或 Telegram Bot 發送時間記錄，例如：「閱讀 AI 書籍 45m #學習」。

2. AI 分類與 Notion 寫入

n8n 攔截訊息並呼叫 AI 拆解出：時間 (45)，內容 (閱讀 AI 書籍) 與標籤 (學習)，並自動寫入 Notion 資料庫中。
